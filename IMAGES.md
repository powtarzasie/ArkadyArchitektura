# Imagery — placeholder manifest & replacement guide

All project photos on the site are **temporary placeholders** (tonally consistent stock,
to be swapped for Katarzyna's real, art-directed project photography later).
Brand assets (logos) and the portrait are real — leave them.

## How to replace a photo (two ways)

**Easiest — overwrite the file:** drop a new image with the **same filename** into `/assets`
(same aspect ratio, see table). Nothing in the code changes. Re-run the OG step if you swap the hero.

**Or — change the path:** every photo reference is centralised in `Arkady Architektura.dc.html`:
- **Project covers + galleries** → the `data()` method (one object per project).
- **Hero image + Instagram feed** → the `heroCover` / `instagram` config in `renderVals()`.
Search the markup for `data-ph="…"` to locate every swappable `<img>` (`hero`, `project`, `gallery`, `instagram`).

> Tip: keep filenames stable so overwriting "just works". If you rename, update `data()` / the config block.

## Notes for real photography
- Shoot/export covers at **4:5** for a consistent portfolio grid; full-width gallery shots can be landscape.
- Keep the warm, calm tonality (cream / greige / brass, soft natural light) for cohesion.
- Add a descriptive `alt` per image when swapping (the `instagram` array already has `alt` fields; project `alt` is built from the title).
- **Performance to-do when real photos land:** export optimised JP/WebP, consider `srcset`/`sizes`,
  and regenerate `og-image.jpg` (1200×630) from the new hero.
- `og-image.jpg` was generated by cover-cropping `img-kitchen-crittall.jpg` to 1200×630; regenerate if the hero changes.
